# Plan 2.1: Project Model & Controller Endpoint

## Objective
Establish the exact API endpoints and MongoDB model required for Managers to securely create and retrieve projects.

## Context
- .gsd/SPEC.md
- .gsd/ARCHITECTURE.md
- backend/src/models/project.model.js
- backend/src/controllers/project.controller.js
- backend/src/routes/project.route.js

## Tasks

<task type="auto">
  <name>Scaffold Project Model</name>
  <files>backend/src/models/project.model.js</files>
  <action>
    Create a robust Mongoose schema representing a Project.
    - Fields needed: name (String, required), description (String), managerId (ObjectId ref User), startDate (Date), deadline (Date).
    - Ensure it safely exports the model using ESModules.
  </action>
  <verify>Check file syntax and ensure no Mongoose Schema errors.</verify>
  <done>Model successfully exported.</done>
</task>

<task type="auto">
  <name>Implement Project Controllers & Router</name>
  <files>
    backend/src/controllers/project.controller.js
    backend/src/routes/project.route.js
  </files>
  <action>
    Implement two heavily guarded endpoints using the existing `verifyJWT` middleware.
    - POST `/` to create a Project (only Managers and Admins). Attaches the `req.user._id` to the project's `managerId`. Use `asyncHandler` and `ApiResponse`.
    - GET `/` to retrieve all projects.
    - Export and attach to the main express `app.use` router in `app.js`.
  </action>
  <verify>Hit the endpoint utilizing postman or browser via localhost:5002</verify>
  <done>POST / and GET / correctly negotiate JWT and MongoDB</done>
</task>

## Success Criteria
- [ ] Mongoose successfully reads the `project.model.js` schema without crashing.
- [ ] The `verifyJWT` successfully prevents non-authenticated users from creating projects.
- [ ] Creating a project attaches the current logged-in user successfully.
